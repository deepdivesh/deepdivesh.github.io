Cross browser UI testing

18 Jun 2017 on javascript

Testing is hard. UI testing is harder. Cross browser UI testing is probably the hardest thing since Flappy Bird.

This is an attempt to make it easier.

The scenario

Before I get too deep into testing, I want to point out that the hardness level of each of the concepts mentioned above is in direct correlation with level of value it delivers to your app - which should not suprise you.

Awesome things are not easy. They are awesome because they are hard.

There are a lot of concepts to grasp when you're doing this. I'll explain how to do it with the scenario I've encountered, while working on Minim.

Minim is an Express application and for testing its UI, the first thing to deal with is Jade templates - we don't have the privilege of working with raw HTML.

Testling is not suitable for HTML and JS testing, check out Zuul and Saucelabs
