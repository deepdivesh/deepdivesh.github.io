You might not need a pagination library

11 Aug 2018 on ruby

Since I use Ruby to build the backend, while having React on the frontend, I've found that adding some sort of pagination library is not worth it.

Those libraries are meant to be used in full-stack Rails apps, while I only need it for the backend layer - if I use Rails, I only use it with the --api switch. I separately deal with problems that those libraries are solving in the view layer, since I have React as a view layer.

That's why I wanted to see how hard would it be to implement my own pagination library. Turns out it's not that hard.

module Pagination
   def self.per_page
     25
   end

   scope :paginate, ->(page){
     offset = [page.to_i - 1, 0].max * per_page
     offset(offset).limit(per_page)
   } do
     def total
       except(:limit, :offset).count(:all)
     end

     def current_page
       page
     end

     def total_pages
       (total.to_f / per_page).ceil
     end

     def more_available?(page)
       page = [page.to_i, 1].max
       page < total_pages
     end
   end
 end

Include this in any Active Record model and you're good to go.

class User < ApplicationRecord
  extend Pagination
end

User.paginate(3)

class User < ApplicationRecord
  scope(:admin), -> { where(:role, :admin) }
end

User.admin.paginate(2)
